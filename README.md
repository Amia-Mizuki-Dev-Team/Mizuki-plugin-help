# Amia-plugin-help

`Amia-plugin-help` 是 Mizuki Bot 的文字帮助前置插件。它在主图片帮助插件处理 `/help` 之前先发送一段简短文字说明，并通过 `block=False` 让同一条帮助指令继续传递给后续插件。

## 当前行为

注册指令：

```text
help
帮助
```

只有在指令没有参数时发送文字提示：

```text
/help
```

以下形式会跳过文字前置，让其他帮助插件自行处理：

```text
/help 7
帮助 economy
```

## 为什么使用 `block=False`

当前插件不是完整帮助菜单，而是帮助链路的前置提示层：

```text
用户发送 help
      ↓
Amia-plugin-help 发送文字说明
      ↓ block=False
后续图片帮助/分类帮助插件继续处理
```

因此处理函数必须使用：

```python
await matcher.send(...)
```

不能使用：

```python
await matcher.finish(...)
```

否则会终止后续帮助插件。

## 当前优先级

```python
priority=1
block=False
```

维护时要确认后续帮助插件的优先级高于 1，否则执行顺序可能变化。

## 当前文本内容

文字提示目前直接写在 `__init__.py` 中，包含：

- Bot 使用入口。
- 群聊说明。
- PJSK 私聊限制。
- qbind 绑定提示。
- 文档/网站入口。

这套实现可运行，但长期维护存在问题：

- 文案修改需要改代码。
- URL 和群号容易过期。
- 无法按环境切换文本。
- 与主帮助菜单可能出现名称不一致。

## 后续建议

Codex 后续可按以下最小改造：

1. 将文字内容移到配置或独立模板文件。
2. 增加 `AMIA_HELP_PREFIX_TEXT` 或列表配置。
3. 将网站地址、交流群号和 qbind 提示拆成独立字段。
4. 启动时校验空文本，不因配置缺失阻断插件加载。
5. 增加无参数发送、有参数跳过、命令链继续三个测试。

不建议立即把所有业务插件帮助文案复制进本仓库。完整功能分类应由主帮助菜单或统一能力索引生成。

## 依赖

```text
nonebot2
nonebot-adapter-onebot（由主项目提供）
```

本插件当前只使用通用 Message 和 CommandArg，不直接依赖数据库或 amia-core。

## 测试建议

应至少验证：

- `/help` 发送前置文字。
- `帮助` 别名可用。
- `/help 7` 不发送前置文字。
- handler 返回后没有阻断后续 matcher。
- 文案中不包含失效 URL 或敏感信息。

## 维护边界

- 保持前置文字简短，不复制整份帮助菜单。
- 常用入口发生变化时同步更新文案。
- 不在代码中写 token、管理员账号或内部地址。
- 不随意修改 `priority` 和 `block`，除非同时验证帮助链路。
- 当前仓库尚未确定公开许可证。
